## 2. 核心组件实现 (Pseudo-Code)

### A. The Obfuscator (造就“必杀”环境)

这是你论文实验设置中最强的**控制变量**。通过调节混淆等级，你可以人为控制 Loss Landscape 的“崎岖程度”和“语义梯度”的信噪比。

```python
# benchmarks/obfuscator.py
import ast

class SemanticStripper(ast.NodeTransformer):
    def __init__(self, level=1):
        self.level = level
        self.mapping = {}
        self.counter = 0

    def _get_new_name(self, original_name):
        if original_name not in self.mapping:
            self.counter += 1
            # Level 1: v_1, v_2 (保留结构，去除语义)
            # Level 2: IlllI, IlIlI (视觉混淆，增加Token难度)
            prefix = "v_" if self.level == 1 else "Il" 
            self.mapping[original_name] = f"{prefix}{self.counter}"
        return self.mapping[original_name]

    def visit_Name(self, node):
        # 核心逻辑：替换所有变量名和函数名
        if isinstance(node.ctx, (ast.Load, ast.Store)):
            node.id = self._get_new_name(node.id)
        return node

    def obfuscate(self, code_str):
        tree = ast.parse(code_str)
        # 1. 移除 Docstrings (去除语义捷径)
        # 2. 重命名变量 (去除命名提示)
        self.visit(tree)
        return ast.unparse(tree)
```

### B. The Momentum Agent (你的新算法)

这是论文的 **Methodology** 核心。

```python
# agent/momentum_agent.py

class SemanticMomentumOptimizer:
    def __init__(self, beta=0.8):
        self.beta = beta
        self.momentum_text = "None. Starting optimization." # m_0

    def step(self, current_code, feedback):
        # 1. 计算梯度 (Calculate Gradient)
        # 让 LLM 分析 Diff 和 Error，提取“修改方向”
        gradient_text = self.llm.analyze_gradient(current_code, feedback)
        # e.g., "Gradient: Fix off-by-one error in loop condition."

        # 2. 更新动量 (Update Momentum)
        # 让 LLM 融合旧动量和新梯度
        self.momentum_text = self.llm.update_momentum(
            prev_momentum=self.momentum_text,
            new_gradient=gradient_text,
            beta=self.beta
        )
        # e.g., "Momentum: Focusing on loop logic. Ignoring variable naming."

        # 3. 执行下降 (Descent Step)
        # 将动量作为“硬约束”注入 Prompt
        new_code = self.llm.edit_code(
            code=current_code,
            feedback=feedback,
            system_prompt=f"CONSTRAINT: Maintain this optimization direction: {self.momentum_text}"
        )
        return new_code
```

### C. The Auditor (动力学审计)

用于量化“震荡”。

```python
# harness/auditor.py
import Levenshtein

class DynamicsAuditor:
    def __init__(self):
        self.history = [] # List of code strings

    def log(self, code):
        self.history.append(code)

    def calculate_oscillation_index(self):
        # 震荡指数 = (总编辑距离 / 净编辑距离) - 1
        # 如果你改过去又改回来，总距离很大，净距离为0，指数 -> 无穷大
        if len(self.history) < 2: return 0
      
        total_dist = sum(Levenshtein.distance(self.history[i], self.history[i+1]) 
                         for i in range(len(self.history)-1))
      
        net_dist = Levenshtein.distance(self.history[0], self.history[-1])
      
        if net_dist == 0: return 10.0 # 完全回到原点
        return (total_dist / net_dist) - 1.0
```

## 3. 实验运行协议 (Run Protocol)

### 实验一：难度压力测试 (The "Stress Test")

**目的**：证明 Baseline 在去语义环境下会崩盘，而 SMO 能挺住。

* **数据集**：HumanEval-DeSemanticized (取前 50 题)。
* **变量 X**：混淆等级 (Level 0: Clean, Level 1: Renamed, Level 2: Aggressive)。
* **对比组**：
  1. **Cursor-style** (Full Context, Temperature=0.0)
  2. **SMO Agent** (Momentum, $\beta=0.7$, Temperature=0.7)
* **预期结果**：
  * Level 0: 两者通过率差不多 (Baseline 甚至略高，因为直接记忆了题目)。
  * Level 1: Baseline 震荡率飙升，通过率跌至 30%。SMO 维持在 60%。
  * Level 2: Baseline 彻底随机游走 (Random Walk)。SMO 仍有收敛能力。

### 实验二：动量参数敏感性 (The "Hyperparam Study")

**目的**：验证 $\beta$ 的物理意义。

* **固定变量**：Level 1 混淆。
* **变量 X**：动量系数 $\beta \in [0.0, 0.3, 0.6, 0.9]$。
  * $\beta=0.0$: 相当于无动量的 Standard Agent。
  * $\beta=0.9$: 相当于“极其固执”的 Agent（惯性太大）。
* **观察指标**：收敛步数 (Steps to Solution)。
* **预期曲线**：U 型曲线。太小震荡，太大过冲 (Overshoot)，中间是 Sweet Spot。
